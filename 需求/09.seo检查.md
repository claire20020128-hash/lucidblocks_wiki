一，先给你一个“优先级用法”（你做站最该盯的 3 层）
1）能不能被收录（先保命）
① Robots Tag / X-Robots-Tag 有没有把页面 noindex 掉
② Canonical（规范页）是不是指错，导致权重被合并到别的 URL ([Google for Developers][1])
③ sitemap.xml / robots.txt 是否可访问（让爬虫更顺）
2）能不能拿到更高点击率（再提效）
① Title / Description 长度与可读性
② SERP 有没有缩略图、站点名、面包屑等视觉元素（直接影响 CTR）([Google for Developers][2])
③ Images 是否有 alt、是否能被谷歌选成预览图（缩略图）([Google for Developers][3])
3）能不能打得过对手（最后比拼）
① Traffic（看盘子大小、对手量级）([Chrome 网上应用店][4])
② Links / 外链与内部链接结构（站内传权）
③ PageSpeed / Core Web Vitals（体验指标，影响整体竞争力）([Google for Developers][5])

二，按左侧菜单逐个说：每一栏你要重点看什么、怎么看、怎么用
1）Overview（最重要的“体检总表”，建议每次必看）
① Title / Description
• 看点：长度是否合理、是否包含主关键词、是否像“给人看”的一句话
• 用法：同一关键词下，优先把“差异点/版本/平台”塞进标题前 55~60 字符内（避免截断）
② Keywords（如果显示 N/A）
• 含义：不是必须项；现在 SEO 不靠 meta keywords 了，重点靠正文与结构化信息
③ Url / Canonical
• 看点：Canonical 是否等于你想收录的那条 URL（尤其 www / 非 www、带不带尾斜杠、参数页）
• 为什么：Canonical 是让搜索引擎在重复内容里选“代表页”的关键提示 ([Google for Developers][1])
④ Robots Tag / X-Robots-Tag
• 看点：有没有 noindex、nofollow、max-image-preview 被限制
• 为什么：robots meta / X-Robots-Tag 会直接控制索引与预览展示 ([Google for Developers][6])
⑤ robots.txt / sitemap.xml
• 看点：是否 Available；点进去能不能打开、是不是 200
⑥ Word Count / Lang / H1-H6
• 看点：Word Count 太少=内容薄；Lang 是否正确；H1 是否为 1 个（很多模板会忘了 H1）
⑦ Images（数量、Unique、Without Alt、Without Title）
• 看点：Without Alt 是否过多（图片 SEO 与可访问性都会受影响）([Google for Developers][3])
⑧ Links（Internal/External、Dofollow/Nofollow）
• 看点：Internal 太少=站内传权弱；External 太多=容易稀释注意力（不是越多越好）

2）Traffic（用来“看盘子/看对手”，不是用来当精准统计）
① 看点
• 访问量/趋势：判断这个站是不是“真有流量”
• Top pages / 关键词分布（如果插件有展示）：用来抄作业找内容结构
② 用法（做新站选词时）
• 只拿它做“相对对比”：A 站明显大于 B 站，说明 SERP 竞争更硬
• 别把某一个具体数字当真；看“量级”和“对比差距”更靠谱 ([Chrome 网上应用店][4])

3）Issues（最适合当“待办清单”）
① 重点盯这三类
• 收录风险：noindex、canonical 错、重定向链/循环
• 内容结构：缺 H1、标题空/重复、描述缺失
• 技术可抓取：SSR/渲染异常（单页应用尤其要注意）
② 用法
• 这里出现红/黄项：优先修“收录类”＞“结构类”＞“体验类”

4）SERP（判断你现在在谷歌上的“展现形态”）
① 重点看
• 有没有缩略图、站点名、面包屑、时间等视觉元素
② 关键点
• 缩略图是谷歌自动选的，但你可以通过图片与元数据去“影响它选哪张”([Google for Developers][3])

5）Density（关键词密度，不要当成 KPI）
① 看点
• 主关键词是否自然出现（标题、H1/H2、首段、图片 alt、内链锚文本）
② 用法
• 只用来发现“完全没覆盖/覆盖过度像堆词”的极端情况

6）Headings（结构是否像“可读的攻略/百科”）
① 看点
• H1 是否 1 个；H2/H3 是否承载模块（Walkthrough、Codes、Tier List 等）
② 用法
• 你的 wiki/攻略站：H2 做大模块，H3 做子问题；这会让内容更容易被理解与复用

7）Images（解决你之前问的“SERP 没缩略图”也常靠它）
① 看点
• 首屏是否有“真实 img”（不是背景图/纯懒加载占位）
• alt 是否完善；图片能否被抓取（200、无防盗链）
② 为什么
• 谷歌会综合多种信号选择预览图；你给到清晰的图片与元数据更容易出图 ([Google for Developers][3])

8）Links（站内结构 + 外链风险）
① 内链（Internal）
• 看点：重要页面有没有被足够多页面链接到（首页/导航/面包屑/相关文章）
② 外链（External）
• 看点：是否过多指向无关站；是否存在可疑跳转链接
③ 用法
• 做 wiki：给“核心页”加更多入口（首页模块、侧边栏、相关推荐）

9）Social（决定你分享到社交平台长什么样，也会反向帮助“选图”）
① 看点
• Open Graph：og:title / og:description / og:image 是否存在且可访问
② 用法
• 给每个核心页准备一张 1200px+ 的 og:image，有助于平台预览，也能影响谷歌对“代表图”的选择 ([Google for Developers][3])

10）Hreflangs（做多语言站必看）
① 看点
• 每个语言版本是否互相指向（A 指 B，B 也要指回 A）
• 是否有 x-default（多地区入口页常用）([Google for Developers][7])
② 为什么
• hreflang 用来告诉谷歌“这些是同内容不同语言/地区版本”，避免互相抢排名 ([Google for Developers][8])

11）Structured（结构化数据，决定你有没有机会拿富结果/更强展示）
① 看点
• 有没有 JSON-LD；Organization / WebSite / Article 等是否齐
• logo / image 是否可访问（很多人写了但图 403）
② 为什么
• 结构化数据帮助谷歌理解页面与实体；Organization 的 logo 还可能影响搜索展示 ([Google for Developers][9])

12）Whois（做竞品/新站判断用）
① 看点
• 域名注册时间、到期时间
② 用法
• 竞品是老域名 + 老站：一般更难打；新域名更可能是“新站窗口期”([Chrome 网上应用店][4])

13）DNS（排查访问/收录异常的“底层”）
① 看点
• 是否正确解析到你的主机；www 与根域是否一致；是否有多余的跳转链
② 用法
• 遇到 301/302 乱跳、https 循环：先看 DNS + SSL 配置再看页面

14）Settings（你自己的偏好设置）
① 用法
• 把你常用的对比数据（比如 Similarweb/Semrush/Ahrefs）固定好，减少来回切工具

15）Archive（看历史版本，做竞品拆解）
① 看点
• 竞品过去怎么写标题/模块/内链；什么时候开始起量

16）Similarweb / Semrush / Ahrefs（外部工具跳转入口）
① 用法
• Similarweb：看渠道结构、国家占比
• Semrush/Ahrefs：看关键词、外链、Top pages
② 注意
• 这些属于“深水区”，你用 AITDK 先筛选“值得深入分析的站”更省时间

17）PageSpeed（体验指标，建议上线前就过一遍）
① 看点（Core Web Vitals）
• LCP（加载速度）、INP（交互响应）、CLS（抖动）([Google for Developers][5])
② 用法
• 你的站是内容站：优先保证首屏大图/主标题快出来、别抖动、JS 别太重

18）Twitter（通常就是 Social 的细分）
① 看点
• twitter:card 是否为 summary_large_image；twitter:image 是否可访问
② 用法
• 方便你检查“分享预览图”为啥不出

三，给你一个“你做游戏站”的固定检查顺序（照着做就行）
1）Overview：先把 Canonical / Robots / sitemap / H1 / Lang 修到没坑
2）Images + Social：把 og:image 与首屏主图做对（为 SERP 缩略图做准备）([Google for Developers][3])
3）Structured + Hreflangs（如果有多语言）：补 Organization logo、hreflang 闭环 ([Google for Developers][10])
4）PageSpeed：首屏速度与抖动先过线 ([Google for Developers][5])
5）Traffic + Links：最后用来做竞品对比和内容扩展方向 ([Chrome 网上应用店][4])
