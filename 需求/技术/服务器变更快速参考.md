# GitHub Token 生成指南

## 🔑 什么是 GitHub Token？

GitHub Personal Access Token (PAT) 是用于访问 GitHub 资源的身份验证凭证，类似于密码，但更安全且可以精细控制权限。

---

## 📋 Token 的作用

在我们的场景中，Token 用于：
1. **从 GHCR 拉取 Docker 镜像**（公开镜像不需要，私有镜像需要）
2. **身份验证**：证明您有权限访问仓库的镜像
3. **安全性**：可以随时撤销，不暴露 GitHub 密码

---

## 🎯 生成 Token 的步骤

### 步骤 1: 访问 GitHub 设置

1. 登录 GitHub
2. 点击右上角头像 → **Settings**
3. 左侧菜单滚动到底部 → **Developer settings**
4. 点击 **Personal access tokens** → **Tokens (classic)**

**直接链接**：https://github.com/settings/tokens

---

### 步骤 2: 创建新 Token

1. 点击 **Generate new token** → **Generate new token (classic)**
2. 填写信息：

| 字段 | 值 | 说明 |
|------|-----|------|
| **Note** | `GHCR Docker Pull` | Token 的名称（自己记得就行） |
| **Expiration** | `No expiration` 或 `90 days` | 过期时间（建议选择不过期） |
| **Scopes** | ✅ `read:packages` | 读取包的权限 |
|  | ✅ `write:packages` | 写入包的权限（可选，服务器只需 read） |

3. 滚动到底部，点击 **Generate token**

---

### 步骤 3: 保存 Token

⚠️ **重要**：Token 只会显示一次！

生成后会看到类似这样的 Token：
```
ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**立即复制并保存到安全的地方**：
- 密码管理器（如 1Password、LastPass）
- 本地加密文件
- 服务器环境变量
