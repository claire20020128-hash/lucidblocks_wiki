cc全部完成后，先检查https://github.com/claire20020128-hash/lucidblocks_wiki/actions，然后在服务器部署，

然后，参考需求\技术\CLOUDFLARE_HTTPS_SETUP.md 去配置Claudeflare：
https://dash.cloudflare.com/610b80b08aa02848902d5fbdafbc9790/lucidblocks.wiki
1、`Cloudflare Dashboard → DNS → Records-Add record`

```
Type: A
Name: www
Target:187.124.74.163
Proxy status: Proxied (橙色云朵) ✅

Type: A
Name: @ (或 域名本身)
Target: 187.124.74.163
Proxy status: Proxied (橙色云朵) ✅
```

2、`Cloudflare Dashboard → SSL/TLS → Overview` 配置加密模式: Full(在我deploy的域名/域名+www 都开启了https的状态下)
3、 `Cloudflare Dashboard → SSL/TLS → Edge Certificates` 配置Always Use HTTPS: 开启 (ON)
4、`Cloudflare Dashboard → Rules → Overview`--"Create rule"-'Redirect Rules'--
Rule name: Redirect non-www to www
If incoming requests match:
● Wildcard pattern
Request URL: https://lucidblocks.wiki/*
Target URL: https://www.lucidblocks.wiki/${1}
Status code: 301
Preserve query string: ✅ 5.检查DNS传播情况：https://www.whatsmydns.net/#A/lucidblocks.wiki

最后，去GSC添加索引并提交站点地图：
https://search.google.com/search-console?resource_id=sc-domain%3Alucidblocks.wiki
验证您的所有权，开始验证，授权，前往资源页面，搜索框搜索域名，测试实际网址，请求编入索引，关闭，
然后在自己的网站地址里https://www.lucidblocks.wiki/sitemap.xml（一定要切换一个VPN ），点击站点地图，再把https://www.lucidblocks.wiki/sitemap.xml 整体复制提交到"添加新的站点地图"，点击提交，成功，关闭。如果无法抓取不要害怕，刷新以下也许就有结果。
