五、4）内容站导航栏分类

下面这 8 个分类，是按“真实玩法系统 + 当前社区问题 + 资料站可持续扩写”来定的。边界我给你切清楚了，避免以后内容互相打架。([Lucid Blocks][1])

1）Guides
边界：只放上手、路线、开局、生存节奏；不放具体配方表，不放纯资料索引。
常见关键词：Lucid Blocks guide；Lucid Blocks first night
改写内容：新手入门、开局 10 分钟该做什么、第一夜怎么活、前期探索顺序。([Steam社区][3])

2）Crafting
边界：只放 Apotheosis / fusion / 配方结果 / 合成逻辑；不放装备评测，不放区域攻略。
常见关键词：Lucid Blocks recipe；Lucid Blocks apotheosis
改写内容：融合系统原理、常用合成结果、实用配方表、材料组合思路。([Lucid Blocks][1])

3）Biomes
边界：只放区域、地形、环境、危险度、资源分布；不放单个怪物条目。
常见关键词：Lucid Blocks Fog Meadow；Lucid Blocks heaven
改写内容：区域地图感知、各 biome 特征、刷资源建议、危险提醒、该带什么工具。([Steam社区][4])

4）Creatures
边界：只放生物、敌人、友方实体、掉落与应对；不放纯地区说明。
常见关键词：Lucid Blocks bubblebear；Lucid Blocks angel
改写内容：怪物/生物图鉴、行为机制、友好或敌对、掉落、应对方式。([Lucid Blocks][1])

5）Items
边界：只放武器、工具、移动道具、功能道具；不放合成总表。
常见关键词：Lucid Blocks hookshot；Lucid Blocks bee glider
改写内容：道具用途、获取方式、什么时候值得做、搭配建议。([Lucid Blocks][1])

6）Achievements
边界：只放 19 个成就与达成路线；不放泛新手教程。
常见关键词：Lucid Blocks achievements；Lucid Blocks achievements guide
改写内容：全成就条件、容易漏掉的点、对应系统入口、100% 路线。([Steam商店][2])

7）Lore
边界：只放世界观、隐藏含义、结局解释、诡异设定；不放纯操作教学。
常见关键词：Lucid Blocks lore；Lucid Blocks ending
改写内容：剧情碎片整理、环境叙事、结局解读、玩家常见理解分歧。([Steam社区][4])

8）Support
边界：只放技术与兼容性问题；不放剧情和资料图鉴。
常见关键词：Lucid Blocks crashing；Lucid Blocks Steam Deck
改写内容：启动报错、Vulkan 问题、Steam Deck 设置、存档位置、画面异常、手柄/键位问题。([Steam商店][2])

六、最后给你一个建站提醒

这个词现在最值钱的，不是“新闻”，也不是“codes”，而是“可检索资料 + 上手问题 + 系统拆解”。所以 lucidblocks.wiki 首页导航最稳的做法，就是把 Crafting、Biomes、Items、Achievements、Support 放到最前面。这样更贴近 Lucid Blocks 当前真实搜索面。([Lucid Blocks][1])
