## 三、SocialBarAd 实现

### 3.1 组件概述

**文件位置**: `src/components/ads/SocialBarAd.tsx`

SocialBarAd 是一个**浮动式社交分享栏广告**，全局加载在页面底部。

### 3.2 路径拼接方式

**是否完全动态路径**: ❌ **否**

**路径拼接方式**: **固定前缀 + 动态 adKey**

```tsx
script.src = `https://pl28666057.effectivegatecpm.com/${scriptPath}.js`;
```

**技术细节**:

- **固定域名**: `pl28666057.effectivegatecpm.com`
- **动态部分**: `${scriptPath}.js`（从环境变量读取）
- **环境变量**: `NEXT_PUBLIC_AD_SOCIAL_BAR`
- **示例**: 如果 `adKey = "abc123"`，则完整路径为 `https://pl28666057.effectivegatecpm.com/abc123.js`

### 3.3 加载位置

**文件位置**: `src/app/[locale]/layout.tsx:133-134`

```tsx
<body suppressHydrationWarning className="antialiased">
  <Analytics />
  <NextIntlClientProvider messages={messages}>
    <ClientBody>{children}</ClientBody>
  </NextIntlClientProvider>
  {/* 社交栏广告 */}
  <SocialBarAd adKey={process.env.NEXT_PUBLIC_AD_SOCIAL_BAR || ""} />
</body>
```

**技术细节**:

- **加载时机**: 页面加载后立即执行
- **作用域**: 全局（所有页面共享）
- **渲染结果**: 组件本身返回 `null`，广告由脚本动态注入 DOM

### 3.4 脚本加载逻辑

```tsx
useEffect(() => {
  if (!adKey || adKey === "0" || scriptLoadedRef.current) return;

  const scriptPath = adKey.trim();

  const script = document.createElement("script");
  script.setAttribute("data-cfasync", "false");
  script.src = `https://pl28666057.effectivegatecpm.com/${scriptPath}.js`;
  script.async = true;
  script.onload = () => {
    console.log("[SocialBarAd] Script loaded:", script.src);
  };
  script.onerror = (error) => {
    console.error("[SocialBarAd] Script failed to load:", error);
  };
  document.body.appendChild(script);
  scriptLoadedRef.current = true;

  return () => {
    if (script.parentNode) {
      script.parentNode.removeChild(script);
    }
    scriptLoadedRef.current = false;
  };
}, [adKey]);
```

**技术细节**:

- **防重复加载**: `scriptLoadedRef.current` 标记
- **异步加载**: `script.async = true`
- **Cloudflare 优化**: `data-cfasync="false"`
- **清理机制**: 组件卸载时移除脚本

## 四、广告组件技术实现

### 4.1 AdBanner 组件

**文件位置**: `src/components/ads/AdBanner.tsx`

#### 支持的广告类型

```typescript
type: "banner-300x250" |
  "banner-468x60" |
  "banner-728x90" |
  "banner-160x600" |
  "banner-320x50";
```

#### 配置映射

```typescript
const AD_CONFIGS = {
  "banner-300x250": { width: 300, height: 250 },
  "banner-468x60": { width: 468, height: 60 },
  "banner-728x90": { width: 728, height: 90 },
  "banner-160x600": { width: 160, height: 600 },
  "banner-320x50": { width: 320, height: 50 },
};
```

#### 动态渲染机制

**队列化加载**:

```typescript
function enqueueHighPerformanceAdLoad(task: () => Promise<void>) {
  const w = window as HighPerformanceWindow;
  const queue = w.__highPerformanceAdQueue ?? Promise.resolve();
  const next = queue.then(task, task);
  w.__highPerformanceAdQueue = next.then(
    () => undefined,
    () => undefined,
  );
  return next;
}
```

**技术细节**:

- **全局队列**: 防止多个广告同时加载导致冲突
- **超时控制**: 8 秒超时（`AD_LOAD_TIMEOUT_MS = 8000`）
- **脚本域名**: `www.highperformanceformat.com`
- **路径格式**: `https://www.highperformanceformat.com/${adKey}/invoke.js`

#### 脚本加载流程

```typescript
const w = window as HighPerformanceWindow;
w.atOptions = {
  key: adKey,
  format: "iframe",
  height: config.height,
  width: config.width,
  params: {},
};

const invokeScript = document.createElement("script");
invokeScript.type = "text/javascript";
invokeScript.src = `https://www.highperformanceformat.com/${adKey}/invoke.js`;
invokeScript.async = true;
container.appendChild(invokeScript);
```

**技术细节**:

- **配置注入**: 通过 `window.atOptions` 传递参数
- **iframe 格式**: 广告以 iframe 形式渲染
- **异步加载**: 不阻塞页面渲染

## 七、关键技术点总结

### 7.1 定位方式对比

| 定位方式 | 使用场景   | 优点                     | 缺点                       |
| -------- | ---------- | ------------------------ | -------------------------- |
| Sticky   | 顶部横幅   | 始终可见，不遮挡内容     | 占用固定高度               |
| Fixed    | 侧边栏     | 滚动时固定，利用空白区域 | 需要计算位置，小屏幕不适用 |
| Static   | 内容中广告 | 自然融入内容流           | 滚动后不可见               |

### 7.2 动态渲染机制

所有广告组件均采用**客户端动态渲染**：

1. **组件挂载**: `useEffect` 触发
2. **脚本创建**: `document.createElement('script')`
3. **队列加载**: 防止并发冲突
4. **超时控制**: 8 秒超时保护
5. **清理机制**: 组件卸载时移除脚本

### 7.3 响应式策略

- **移动端**: 仅显示 320×50 横幅 + 原生广告
- **平板端**: 显示所有横幅广告
- **桌面端**: 显示所有广告（包括侧边栏）

### 7.4 性能优化

- **异步加载**: 所有脚本使用 `async`
- **队列化**: 防止多个广告同时加载
- **懒加载**: 广告仅在组件挂载时加载
- **防重复**: `scriptLoadedRef` 标记防止重复加载

## 八、常见问题

### Q1: 广告不显示？

**检查清单**:

1. 环境变量是否正确配置
2. 广告键值是否有效（非 `0` 或空字符串）
3. 浏览器控制台是否有脚本加载错误
4. 广告拦截插件是否启用

### Q2: 侧边栏广告位置不对？

**原因**: 屏幕宽度不足或计算公式不匹配

**解决方案**:

- 确保屏幕宽度 ≥ 1280px
- 调整 `calc()` 公式中的 `1280px` 为实际内容区宽度

### Q3: SocialBarAd 脚本路径错误？

**检查**:

- 环境变量 `NEXT_PUBLIC_AD_SOCIAL_BAR` 是否正确
- 脚本路径是否需要包含子目录（如 `path/to/script`）

### Q4: 广告加载超时？

**原因**: 网络慢或广告服务器响应慢

**解决方案**:

- 调整 `AD_LOAD_TIMEOUT_MS` 值（默认 8000ms）
- 检查广告服务器可用性
