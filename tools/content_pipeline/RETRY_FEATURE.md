# 收集重试功能说明

## 功能概述

`collect.py` 现在支持智能重试功能，可以自动检测已有结果并只重新搜索失败的关键词，节省 API 配额和时间。

**新增功能**：实时重试机制 - 对于搜索失败的关键词，自动重试最多3次（可配置）。

## 工作原理

### 自动检测
每次运行 `collect.py` 时，会自动：
1. 检查 `out/pending_review.json` 是否存在
2. 加载已有的搜索结果
3. 分析每个关键词的 YouTube 和 Web 结果
4. 只重新搜索缺失的部分

### 判断逻辑

**YouTube 和 Web 独立判断**：
- 如果某关键词的 `youtube.count > 0` 且有 items → 跳过 YouTube 搜索
- 如果某关键词的 `web.count > 0` 且有 items → 跳过 Web 搜索
- 如果某关键词只缺少 Web → 只搜索 Web，不搜索 YouTube
- 如果某关键词只缺少 YouTube → 只搜索 YouTube，不搜索 Web

### 实时重试机制

**每次运行时自动重试**：
- 对于需要搜索的关键词，如果第一次搜索失败（返回空结果），会自动重试
- 默认最多重试 3 次（可通过配置修改）
- 每次重试之间等待 2 秒（可通过配置修改）
- YouTube 和 Web 的重试是独立的，互不影响
- 重试次数不记录到文件，每次运行都是独立的

### 结果合并
- 如果有新结果，使用新结果（可以更新过时的数据）
- 如果没有新结果，保留已有结果
- 所有关键词都会出现在最终的 `pending_review.json` 中

## 使用方法

### 配置参数

可以通过 `.env` 文件配置重试行为：

```bash
# 搜索重试配置
SEARCH_MAX_RETRIES=3      # 最大重试次数（默认3次）
SEARCH_RETRY_DELAY=2      # 重试间隔秒数（默认2秒）
```

如果不设置这些参数，将使用默认值。

### 正常使用（自动重试）
```bash
python3 collect.py --json ../keywords.json
```

**无需任何额外参数**，重试逻辑会自动运行。

### 强制重新搜索所有关键词
如果想重新搜索所有关键词，只需删除已有结果：
```bash
rm out/pending_review.json
python3 collect.py --json ../keywords.json
```

## 优势

1. **节省 API 配额**：不重复搜索已成功的关键词
2. **节省时间**：只重试失败的部分
3. **自动重试**：每次运行时自动重试失败的关键词，最多3次
4. **用户友好**：无需手动管理，自动检测和重试
5. **保留已有结果**：不会丢失已成功收集的数据
6. **透明度**：打印详细的重试过程和统计信息
7. **独立重试**：YouTube 和 Web 独立重试，互不影响
8. **可配置**：可通过环境变量调整重试次数和间隔

## 测试场景

### 场景 1: 部分 Web 搜索失败
```
关键词 A: YouTube ✓ (5个), Web ✗ (0个)
关键词 B: YouTube ✓ (3个), Web ✓ (4个)
关键词 C: YouTube ✓ (2个), Web ✗ (0个)

重试时：
- 只搜索关键词 A 和 C 的 Web
- 不搜索任何 YouTube
- 保留所有已有的 YouTube 结果
```

### 场景 2: 部分 YouTube 搜索失败
```
关键词 A: YouTube ✗ (0个), Web ✓ (6个)
关键词 B: YouTube ✓ (4个), Web ✓ (3个)

重试时：
- 只搜索关键词 A 的 YouTube
- 不搜索任何 Web
- 保留所有已有的 Web 结果
```

### 场景 3: 混合失败
```
关键词 A: YouTube ✓ (5个), Web ✗ (0个)
关键词 B: YouTube ✗ (0个), Web ✓ (4个)
关键词 C: YouTube ✗ (0个), Web ✗ (0个)

重试时：
- 搜索关键词 A 的 Web
- 搜索关键词 B 的 YouTube
- 搜索关键词 C 的 YouTube 和 Web
```

## 技术实现

### 新增函数

1. **`load_existing_results()`**
   - 加载 `pending_review.json`
   - 构建关键词到结果的映射
   - 错误处理：文件不存在或格式错误时返回空字典

2. **`filter_keywords_for_retry()`**
   - 独立判断 YouTube 和 Web 是否需要重试
   - 返回两个独立的重试列表
   - 支持部分重试（只重试失败的部分）

3. **`merge_results()`**
   - 合并新旧结果
   - 优先使用新结果（如果有）
   - 保留所有关键词的完整数据

4. **`search_with_retry()`** ⭐ 新增
   - 带重试机制的搜索函数
   - 接收搜索函数、关键词列表、数据源名称
   - 实现最多 N 次重试逻辑（从配置读取）
   - 每次重试间隔 M 秒（从配置读取）
   - 打印详细的重试过程
   - 返回成功搜索的结果字典

### 配置参数

在 `core/config.py` 中新增：
```python
# 搜索重试配置
SEARCH_MAX_RETRIES = int(os.getenv("SEARCH_MAX_RETRIES", "3"))
SEARCH_RETRY_DELAY = int(os.getenv("SEARCH_RETRY_DELAY", "2"))
```

### 修改的流程

```
加载关键词
  ↓
加载已有结果
  ↓
过滤需要重试的关键词
  ↓
打印统计信息
  ↓
并行搜索（只搜索需要重试的，带自动重试）
  ├─ YouTube: search_with_retry() - 最多重试3次
  └─ Web: search_with_retry() - 最多重试3次
  ↓
合并新旧结果
  ↓
保存到 pending_review.json
```

## 注意事项

1. **向后兼容**：如果 `pending_review.json` 不存在或格式错误，会回退到正常搜索所有关键词
2. **数据完整性**：合并逻辑确保不会丢失任何已有数据
3. **透明度**：详细的统计信息和重试过程让用户了解发生了什么
4. **灵活性**：可以通过删除 `pending_review.json` 强制重新搜索
5. **实时重试**：重试次数不记录到文件，每次运行都是独立的
6. **避免限流**：重试间隔可配置，默认2秒，避免 API 限流
7. **独立重试**：YouTube 和 Web 的重试是完全独立的，互不影响
