# 首屏图片优化方案

## 问题定位

hero.png 在 Vercel 无法显示，排查结论：
- 分辨率：768 × 432
- 体积：388 KB（不算大）
- 原因：**路径或 Next.js 配置错误**导致 404，非文件过大

## 优化结果

| 项目 | 优化前 | 优化后 | 改善 |
|------|--------|--------|------|
| 图片格式 | PNG | WebP | 现代格式 |

| 加载优先级 | `priority` | `priority` | 保持 |

* 你看到的 **“Image Optimization – Transformations 5.1 K / 5 K”** 表示免费额度（5 000 次/月）已被用尽，新请求会继续计数并被标黄提示，可能触发付费或被暂时限流；

* 免费 Hobby 计划每月仅 5 K 转换，长期运营站点建议：①上传预压缩 WebP、②开启静态导出、③或升级到 Pro。

## 1. 了解「Exceeded free resources」提示

| 指标                                        | 免费 Hobby 额度   | 你当前值         | 说明                                |
| ----------------------------------------- | ------------- | ------------ | --------------------------------- |
| Image Optimization – Transformations      | 5 000/月       | 5 100+       | 每一次不同尺寸、格式的处理都会消耗额度 ([Vercel][1]) |
| Image Optimization – Cache Writes / Reads | 100 K / 300 K | 5.9 K / 13 K | 命中 CDN 缓存前后写/读次数 ([Vercel][2])    |

**为什么突然超额？**

* 首页或列表页图片尺寸多、格式杂，Next.js 的 `<Image>` 组件会在首屏自动请求多种 breakpoints；
* 文章数量一旦增多，Googlebot 抓取也会触发转换；
* 测试期间多次清缓存 / 重新部署，旧缓存失效又重新计算。

### 优化思路（举例）

1. **本地预处理**：上传 1280 px 以内的 WebP/JPEG，小于 200 KB。
2. **关掉自动优化**：在 `next.config.js` 里

   ```js
   module.exports = { images: { unoptimized: true } }
   ```
3. **自定义 Loader**：指向【wsrv.nl】或 Cloudflare Images，示例：

   ```js
   export const myLoader = ({src, width, quality}) =>
     `https://wsrv.nl/?url=${src}&w=${width}&q=${quality||75}`  
   ```

   然后在 `next/image` 里 `loader={myLoader}` 使用，避免计入 Vercel 配额 ([Reddit][3])。

## 2. 删除 Vercel 项目的两种方法

### 2.1 图形界面（Dashboard）

1. 登录【Vercel Dashboard】，点进待删项目。
2. 左侧侧边栏选择 **Settings** → **General**。
3. 滑动到最底部 **Danger Zone** 区块，点击 **Delete Project**。
4. 在弹窗内手动输入完整项目名确认 → **Continue** → **Delete**。

   * 系统会同步删除：所有历史部署、环境变量、Webhook 等 ([Vercel][4])。

> **演示场景**：
> 你需要清理临时测试站 `deadlydelivery-net`.vercel.app——
> 打开项目 → Settings → Danger Zone → Delete → 输入 `deadlydelivery-net` → Continue → Done。几秒钟内域名解除占用，可重新绑定到新仓库。

### 2.2 命令行（CLI）

```bash
npm i -g vercel
vercel rm totalchaos.info
# 交互式确认后即可删除
```

* `vercel rm` 默认为删除部署；加 `--safe --yes` 可跳过确认；加 `-p` 删除整个项目 ([Vercel][5])。
* CLI 适合批量脚本，或 CI 清理旧环境。

## 4. 为什么搜索结果没有显示 favicon？

* Google 仅在 **优质搜索卡片**（Rich Result）中展示站点图标；检测条件之一是提供 `rel="icon"` 或 `rel="apple-touch-icon"` 并 **保证尺寸 48×48 – 192×192 的 PNG/SVG**。
* 32×32 只是传统浏览器 Tab 要求，SERP 更偏好 48 px+ 且透明背景。
* 在公共路径 `/favicon.ico` 同时提供 16×16、32×32，多尺寸 ICO 也无妨，但确保在 `<link rel="icon" href="/favicon.ico" sizes="any">` 之外，再添加

  ```html
  <link rel="icon" href="/icon-192.png" type="image/png" sizes="192x192">
  ```
* 更新后到【Google Search Console】→ **外观 → 站点图标** 请求重新抓取，通常几天内生效 ([Vercel][6])。
