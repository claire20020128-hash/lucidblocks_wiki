# 翻译脚本问题修复说明

## 你遇到的三个问题

### 问题 1：路径解析错误 ❌

**现象**：
```
脚本在 tools/articles/modules/transpage/ 目录下运行
但相对路径 src/locales/ 会从当前目录查找，而不是项目根目录
```

**原因**：
```python
# 错误的路径解析
self.messages_dir = Path('src/locales/')  # 从当前目录查找
```

**修复**：
```python
# 正确的路径解析
def find_project_root():
    """查找项目根目录（包含 package.json 的目录）"""
    current = Path(script_dir)
    for _ in range(5):
        package_json = current / 'package.json'
        if package_json.exists():
            return current
        current = current.parent
    return Path(script_dir).parent.parent.parent.parent

project_root = find_project_root()
locales_dir = project_root / 'src' / 'locales'
```

### 问题 2：API 响应解析错误 ❌

**现象**：
```python
# 第 157 行
data = await response.json()  # 假设响应是 JSON
```

**原因**：
- 某些 API 返回流式响应（`text/event-stream`）
- 直接使用 `response.json()` 会失败

**修复**：
创建了 `api_response_handler.py` 模块，支持两种响应格式：

```python
async def parse_api_response(response, target_lang):
    """自动检测并解析响应类型"""
    content_type = response.headers.get('Content-Type', '')

    if 'text/event-stream' in content_type:
        return await parse_stream_response(response, target_lang)
    else:
        return await parse_json_response(response, target_lang)
```

---

### 问题 3：Windows 编码问题 ❌

**现象**：
```
Windows 控制台默认使用 GBK 编码
UTF-8 字符显示乱码或报错
无法保存包含中文的文件
```

**原因**：
- Windows 默认代码页是 936（GBK）
- Python 默认使用系统编码

**修复**：
```python
# 在脚本开头添加
# -*- coding: utf-8 -*-

import sys
import os
import io

if sys.platform == 'win32':
    # 强制使用 UTF-8 编码
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', errors='replace')
    sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8', errors='replace')
    # 设置控制台代码页为 UTF-8
    os.system('chcp 65001 > nul 2>&1')
```
