# 翻译脚本使用说明

## 问题分析

### 你遇到的问题

**错误命令**：
```bash
python translate-messages-enhanced.py --all --overwrite
```

**问题原因**：
1. ❌ `--all` 参数不存在（脚本不支持这个参数）
2. ❌ 配置文件中的语言列表与 `routing.ts` 不匹配
3. ❌ 脚本在遇到未知参数时可能进入未定义状态

### 根本原因

`translate-messages-enhanced.py` 支持的参数：
- `--lang` - 指定目标语言（逗号分隔）
- `--overwrite` - 覆盖已有文件
- `--incremental` - 增量翻译
- `--strategy` - 分块策略
- `--report` - 生成报告

**没有 `--all` 参数！**

---

## 解决方案

### 方案 1：使用新脚本（推荐）✅

我创建了一个新脚本 `translate-all-locales.py`，它会：
- ✅ 自动从 `src/i18n/routing.ts` 读取语言列表
- ✅ 避免手动输入错误
- ✅ 确保配置与项目一致

**使用方法**：

```bash
# 进入脚本目录
cd tools/articles/modules/transpage

# 翻译所有语言（自动从 routing.ts 读取）
python translate-all-locales.py --overwrite

# 只翻译指定语言
python translate-all-locales.py --lang es,pt --overwrite

# 不覆盖已有文件（跳过已存在的）
python translate-all-locales.py
```

### 方案 2：使用原脚本（手动指定）

如果你想继续使用 `translate-messages-enhanced.py`：

```bash
# 方式 1：明确指定所有语言
python translate-messages-enhanced.py --lang es,pt,ja,ru,de,fr,tr --overwrite

# 方式 2：不指定 --lang，使用配置文件默认列表
python translate-messages-enhanced.py --overwrite
```

## 常见问题

### Q1: 翻译卡住了怎么办？

**原因**：
- 使用了不存在的参数（如 `--all`）
- 网络问题导致 API 调用超时
- 文件太大导致 token 限制

**解决**：
1. 按 `Ctrl+C` 终止进程
2. 检查命令参数是否正确
3. 使用新脚本重新尝试

### Q2: 如何只翻译部分语言？

```bash
# 只翻译西班牙语和葡萄牙语
python translate-all-locales.py --lang es,pt --overwrite
```

### Q3: 如何避免覆盖已有翻译？

```bash
# 不加 --overwrite 参数
python translate-all-locales.py
```

### Q4: 翻译失败了怎么办？

**检查清单**：
1. ✅ API 密钥是否正确（`transpage_config.json`）
2. ✅ 网络连接是否正常
3. ✅ `en.json` 文件是否存在且格式正确
4. ✅ Python 依赖是否安装（`aiohttp`）

**安装依赖**：
```bash
pip install aiohttp
```
