## 脚本对比

### `translate-all-locales.py`（新脚本）

**优点**：
- ✅ 自动读取 `routing.ts` 语言列表
- ✅ 简单易用，参数少
- ✅ 不容易出错
- ✅ 适合 AI 助手

**缺点**：
- ❌ 功能相对简单
- ❌ 不支持增量翻译

**适用场景**：
- 首次翻译所有语言
- 完全重新翻译
- 快速批量翻译

### `translate-messages-enhanced.py`（原脚本）

**优点**：
- ✅ 功能强大（增量翻译、断点续传）
- ✅ 支持多种分块策略
- ✅ 详细的验证和报告

**缺点**：
- ❌ 参数复杂
- ❌ 需要手动指定语言
- ❌ 容易输错参数

**适用场景**：
- 增量翻译（只翻译变化部分）
- 需要断点续传
- 需要详细报告

## 推荐使用方式

### 场景 1：首次翻译或完全重新翻译

**推荐**：使用新脚本
```bash
cd tools/articles/modules/transpage
python translate-all-locales.py --overwrite
```

### 场景 2：只翻译部分语言

**推荐**：使用新脚本
```bash
cd tools/articles/modules/transpage

```

### 场景 3：增量翻译（只翻译变化部分）

**推荐**：使用原脚本
```bash

python translate-messages-enhanced.py --incremental
```

### 场景 4：需要详细报告

**推荐**：使用原脚本
```bash
cd tools/articles/modules/transpage
python translate-messages-enhanced.py --lang es,pt --report
```

## 常见问题

### Q: 为什么不删除旧脚本？

**答**：
1. 旧脚本功能更强大（增量翻译、断点续传）
2. 新脚本实际上是对旧脚本的封装
3. 某些场景下旧脚本更合适

### Q: 两个脚本有什么关系？

**答**：
```
translate-all-locales.py (新脚本)
    └── 调用 translate-messages-enhanced.py 的核心模块
        ├── translator.py
        ├── smart_chunk_translator.py
        ├── checkpoint_manager.py
        └── enhanced_validator.py
```

### Q: 如何选择使用哪个脚本？

**答**：
- 简单场景 → 使用新脚本 `translate-all-locales.py`
- 复杂场景 → 使用旧脚本 `translate-messages-enhanced.py`
- 不确定 → 使用新脚本
