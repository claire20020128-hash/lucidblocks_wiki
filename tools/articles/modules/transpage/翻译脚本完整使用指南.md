## 🔧 故障排查

### 问题 1：脚本卡住，没有输出

**检查**：
```bash
# 使用 -u 参数强制无缓冲
python -u translate-all-locales-ultimate.py --overwrite
```

**原因**：Python 输出缓冲
**解决**：ultimate 版本已内置修复

### 问题 2：API 超时

**现象**：
```
[TIME] [ES] 超时 (尝试 1/8)
```

**原因**：内容太大或网络慢
**解决**：
- ultimate 版本已增加超时时间（3600秒）
- 使用智能分块翻译（自动拆分大内容）

### 问题 3：路径错误

**现象**：
```
[错误] 找不到 src/locales/en.json
```

**原因**：脚本无法找到项目根目录
**解决**：
- 确认 `package.json` 存在于项目根目录
- ultimate 版本会显示 `[项目根目录] ...`，检查是否正确

### 问题 4：Windows 乱码

**现象**：控制台显示乱码

**解决**：
- ultimate 版本已内置 UTF-8 修复
- 如果还有问题，手动设置：
```bash
chcp 65001
python translate-all-locales-ultimate.py --overwrite
```

## 📊 性能优化

### 智能分块策略

ultimate 版本使用 `smart_chunk_translator.py`，自动按以下策略分块：

1. **top_level**（默认）- 按顶层键分块（如 `seo`, `nav`, `modules`）
2. **medium** - 30个键一块
3. **small** - 15个键一块
4. **tiny** - 5个键一块（最后手段）

如果大块失败，自动降级到小块重试。

### API 限流保护

每个语言之间延迟2秒，避免触发 API 限流：
```python
if idx < len(target_langs):
    print_flush(f"[等待] 2秒后继续下一个语言...")
    await asyncio.sleep(2)
```

## 🎓 从经验中学到的教训

### 1. 输出缓冲很重要
**教训**：Python 默认缓冲输出，导致看不到实时进度
**解决**：`line_buffering=True` + `flush=True`

### 2. 超时配置要合理
**教训**：默认超时太短，大文件会失败
**解决**：增加到3600秒，并分别配置连接和读取超时

### 3. 进度显示提升体验
**教训**：不知道脚本是否在运行，容易误以为卡住
**解决**：添加详细的进度显示

### 4. API 限流要注意
**教训**：并发请求太多可能被封
**解决**：逐个翻译，每个之间延迟2秒
