## 新建项目代码上传流程（无需重复认证）

### 前提条件（一次性配置）

1. **SSH 密钥已配置**
   - 公钥已添加到 GitHub 账号：https://github.com/settings/keys
   - 私钥存储在本机：`~/.ssh/id_ed25519` 或 `~/.ssh/id_rsa`

2. **SSH 配置文件已设置（解决端口22被封问题）**
   ```bash
   # 编辑 ~/.ssh/config 文件，添加以下内容：
   Host github.com
     Hostname ssh.github.com
     Port 443
     User git
   ```

3. **验证 SSH 连接**
   ```bash
   ssh -T git@github.com
   # 应该看到：Hi username! You've successfully authenticated...
   ```

### 新项目上传步骤

#### 1. 在 GitHub 创建新仓库
- 访问：https://github.com/new
- 填写仓库名（如：my-new-game-site）
- 选择 Private 或 Public
- **不要**勾选 "Add README"、"Add .gitignore" 等选项
- 点击 "Create repository"

#### 2. 本地初始化并上传

```bash
# 进入项目目录
cd /path/to/your/project

# 初始化 Git 仓库
git init

# 添加所有文件到暂存区
git add .

# 创建初始提交
git commit -m "Initial commit: Project setup

- Add project files
- Configure dependencies
- Set up build scripts

🤖 Generated with [Claude Code](https://claude.com/claude-code)

Co-Authored-By: Claude <noreply@anthropic.com>"

# 添加远程仓库（使用 SSH 格式，不是 HTTPS）
git remote add origin git@github.com:libin257/my-new-game-site.git

# 推送到远程仓库
git push -u origin main
```

### 关键点说明

✅ **使用 SSH URL 格式**
```bash
# ✅ 正确（SSH 格式）- 无需每次认证
git@github.com:username/repo.git

# ❌ 错误（HTTPS 格式）- 需要输入 token
https://github.com/username/repo.git
```

✅ **为什么不需要重复认证？**
- SSH 公钥已绑定到你的 GitHub 账号（账号级权限）
- 对自己名下的所有仓库都有读写权限
- SSH 密钥会自动被 Keychain 管理，无需手动输入密码

✅ **端口 443 配置的作用**
- 解决某些网络环境下 22 端口被封的问题
- 通过 HTTPS 端口（443）建立 SSH 连接
- 一次配置，永久有效

### 常见问题排查

#### 问题1：提示 "Repository not found"
```bash
# 检查远程 URL 格式
git remote -v

# 如果是 HTTPS 格式，改为 SSH
git remote set-url origin git@github.com:username/repo.git
```

#### 问题2：提示 "Permission denied"
```bash
# 检查 SSH 连接
ssh -T git@github.com

# 如果失败，检查 SSH 密钥
ls -la ~/.ssh/
ssh-add -l

# 添加密钥到 agent
ssh-add ~/.ssh/id_ed25519
```

#### 问题3：端口 22 连接失败
```bash
# 确认 ~/.ssh/config 已配置 443 端口
cat ~/.ssh/config | grep -A 3 "Host github.com"

# 测试 443 端口连接
ssh -T -p 443 git@ssh.github.com
```

### 总结：一次配置，终身受益

1. **SSH 密钥** → 添加到 GitHub 账号（一次性）
2. **SSH config** → 配置 443 端口（一次性）
3. **新项目** → 直接用 `git@github.com:...` 格式推送
4. **无需认证** → 自动使用 SSH 密钥验证

这样，每次新建项目都只需要：
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin git@github.com:username/new-repo.git
git push -u origin main
```
